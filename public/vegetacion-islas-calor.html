🌳 Vegetación e Islas de Calor Urbano

ℹ️ Acerca de esta herramienta: Esta aplicación integra datos de Sentinel-2 (NDVI), Landsat 8/9, MODIS (LST) y GHSL (población y urbanización) para identificar islas de calor, brechas de vegetación y priorizar distritos en Lima. Los datos se actualizan mensualmente.

📖 Metodología

NDVI: Sentinel-2 (10m) + Landsat 8/9 (30m), compuesto mensual (mediana)

LST: MODIS MOD11A2 (1km, 8 días), anomalía respecto climatología 2018-2022

PRIORIDAD: z(LST_anom) - z(NDVI), ponderado por población (GHSL)

SMOD: GHSL P2023A (100m), clasifica grado de urbanización

🚀 Guía de Implementación Completa

📋 Pasos para Implementar en Google Earth Engine
  1. Abre el Google Earth Engine Code Editor
  2. Crea un nuevo script (File → New → Repository → New File)
  3. Copia y pega el código JavaScript completo proporcionado en la documentación
  4. Opcionalmente, carga tu propio asset de distritos de Lima (Format: FeatureCollection)
  5. Ejecuta el script (Run button) - la aplicación se cargará en el Code Editor
  6. Para publicar como app web: Apps → Publish → New App
  7. Configura permisos y comparte el enlace público
💡 Datasets Utilizados (IDs de GEE):
  • COPERNICUS/S2_SR_HARMONIZED - Sentinel-2 Surface Reflectance
  • LANDSAT/LC08/C02/T1_L2 - Landsat 8 Collection 2 Tier 1 Level 2
  • LANDSAT/LC09/C02/T1_L2 - Landsat 9 Collection 2 Tier 1 Level 2
  • MODIS/061/MOD11A2 - MODIS Land Surface Temperature (8-day, 1km)
  • JRC/GHSL/P2023A/GHS_POP/2020 - Global Human Settlement Layer Population
  • JRC/GHSL/P2023A/GHS_SMOD_V2-0/2020 - Settlement Model Grid
  • FAO/GAUL/2015/level2 - Global Administrative Unit Layers (fallback)
